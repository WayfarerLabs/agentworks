# Mise Integration - High-Level Architecture

## Overview

Mise is added as a first-class package manager in agentworks. It is installed system-wide via apt
and then configured per-user. The integrity story delegates entirely to mise's own lockfile
mechanism: users generate lockfiles with `mise lock` and bring them to agentworks via config paths
or dotfiles.

There is no agentworks-specific catalog for mise packages. Tool declarations are either inline
`name@version` strings in the agentworks config, or come from the user's dotfiles (which may
include a `~/.config/mise/config.toml`). Lockfiles are user-provided, not generated by agentworks.

## Component Overview

```
agentworks config (TOML)
    |
    |  mise_packages = ["jq@1.8.1", "adr-tools@3.0.0"]
    |  mise_lockfile = "git::https://github.com/user/infra.git//mise.lock"
    |
    v
initializer.py (VM init / reinit)
    |
    |  1. Install mise via apt + /etc/profile.d/mise.sh (system-wide)
    |  2. Write ~/.config/mise/config.toml from mise_packages (if set)
    |  3. Sync dotfiles (may override mise config, may provide lockfile)
    |  4. Copy/clone lockfile to ~/.config/mise/mise.lock (if set)
    |  5. Run: mise install --locked (or unlocked fallback)
    |  6. Add ~/.local/share/mise/shims to PATH
    |
    v
mise (on VM)
    |
    |  Installs tools, verifies checksums against lockfile
    |
    v
~/.local/share/mise/installs/...  (tool binaries)
```

## Mise Installation via Apt

Unlike other third-party apt packages in the catalog (GitHub CLI, HashiCorp, etc.) which are opt-in
user selections, mise is agentworks infrastructure. Its installation is hardcoded in the initializer
rather than represented as catalog entries.

When `install_mise = true` (the default), the initializer performs these steps as root during init:

1. Download the GPG key from `https://mise.jdx.dev/gpg-key.pub` to
   `/etc/apt/keyrings/mise-archive-keyring.asc`.
2. Write the apt source list to `/etc/apt/sources.list.d/mise.list`:
   `deb [signed-by=/etc/apt/keyrings/mise-archive-keyring.asc] https://mise.jdx.dev/deb stable main`
3. Run `apt update` and `apt install -y mise`.

This installs the mise binary to `/usr/bin/mise`, available to all users immediately. The apt
repository supports both amd64 and arm64 architectures. The step is idempotent.

Shell activation (`eval "$(mise activate <shell>)"`) is per-user, controlled by `mise_activate`
(default true). When enabled, activation lines are added to `~/.agentworks-path.sh` which is
sourced by both `~/.profile` (bash) and `~/.zprofile` (zsh). The activation detects the running
shell via `$ZSH_VERSION` / `$BASH_VERSION` and emits a warning for unsupported shells.

## Config Surface

### VM config (system-level + admin user)

```toml
[vm.config]
# -- System-level --
install_mise = true                              # install mise via apt (default: true)

# -- Admin user mise settings --
# mise_packages = ["jq@1.8.1", "adr-tools@3.0.0"]  # optional tool declarations
# mise_lockfile = "~/.config/agentworks/mise.lock"   # local path, or git:: source ref
# mise_allow_unlocked = false                        # allow install without lockfile coverage
# mise_install_before = "7d"                         # reject versions newer than this
```

### Agent config

```toml
[agent.config]
# mise_packages = ["jq@1.8.1"]                      # optional tool declarations
# mise_lockfile = "~/.config/agentworks/agent-mise.lock"
# mise_allow_unlocked = false
# mise_install_before = "7d"
```

Agents default to no mise packages, no lockfile. They only get tools if explicitly configured.

## Installation Flow

### Sequencing (admin user during init)

The ordering of mise-related steps within `_phase_b_setup()` is:

1. Apt packages, snap, system install commands (existing steps)
2. **Mise apt install** (system-wide binary + `/etc/profile.d/mise.sh`)
3. **Mise config write** (`mise_packages` + `mise_install_before` to `~/.config/mise/config.toml`)
4. Git credentials (existing step, needed before lockfile repo clone)
5. **Dotfiles** (may override mise config, may provide lockfile)
6. **Mise lockfile** (copy from `mise_lockfile` path or clone `mise_lockfile_repo`, overwriting
   any dotfiles-provided lockfile if explicitly configured)
7. **Mise install** (locked or unlocked, see below)
8. User install commands (existing step, may depend on mise-installed tools)

This ordering ensures:
- Agentworks-written mise config is in place before dotfiles, so dotfiles can override it.
- Git credentials are available before lockfile repo clone.
- Dotfiles can provide their own mise config and/or lockfile.
- An explicit `mise_lockfile` / `mise_lockfile_repo` takes precedence over dotfiles-provided
  lockfiles.
- Mise tools are installed before user install commands that may depend on them.

### Per-user setup steps

1. **Add shims to PATH**: Add `~/.local/share/mise/shims` to `~/.agentworks-path.sh`.

2. **Write mise config** (if `mise_packages` is set): Generate `~/.config/mise/config.toml` with
   tool declarations and the `install_before` setting.

   ```toml
   [settings]
   install_before = "7d"

   [tools]
   "jq" = "1.8.1"
   "adr-tools" = "3.0.0"
   ```

3. **Fetch lockfile**: If `mise_lockfile` is set, resolve the source reference (local path or
   `git::` URL) and copy the file to `~/.config/mise/mise.lock`. See source-ref-lld.md for the
   source reference syntax. This step runs after dotfiles, so an explicit lockfile config overrides
   any dotfiles-provided lockfile.

4. **Run mise install**: Detect whether a lockfile is present at `~/.config/mise/mise.lock`
   (from any source: explicit config, dotfiles, or lockfile repo).
   - If lockfile present: run `mise install --locked`.
     - If this fails because some packages are not in the lockfile:
       - If `mise_allow_unlocked = false` (default): log the error, surface which packages are
         missing from the lockfile.
       - If `mise_allow_unlocked = true`: warn about the unlocked packages, re-run `mise install`
         (without `--locked`) to install them anyway.
   - If no lockfile present: run `mise install` (unlocked).

### Idempotency

All steps are idempotent. Apt skips already-installed packages. `mise install` is a no-op if tools
are already present at the correct versions. This aligns with existing reinit behavior.

## Validation

Config loading validates:

- `mise_packages` entries match the `name@version` format.
- `mise_lockfile` (if set) is a valid source reference (local path or `git::` URL).
  Validated at config load time via `parse_source_ref()`.
- `mise_install_before` is a valid mise duration string (e.g., `"7d"`, `"90d"`, `"2024-06-01"`).

## Error Handling

Mise installation failures are non-fatal, matching the existing installer pattern:

- If mise fails to install via apt, a warning is logged and mise per-user steps are skipped.
- If `mise install --locked` fails due to unlocked packages and `mise_allow_unlocked = false`,
  a warning is logged identifying the missing packages.
- If `mise install` fails entirely, a warning is logged.
- The VM receives `PARTIAL` init status if any warnings occur.
- All failures are captured in the SSH logger for later inspection via `agentworks vm logs`.
